إعداد Firebase Push Notifications

دليل تفاعلي خطوة بخطوة لإضافة الإشعارات الفورية لتطبيق نواقص

💡 لماذا Firebase؟ — الإشعارات الفورية تتيح للمستخدمين تلقي تنبيهات فورية عند:
  • 💬 وصول رسالة جديدة
  • ❤️ إعجاب أو تعليق على منشوراتهم
  • 📢 عروض خاصة من السوق
  • 💰 تأكيد شحن المحفظة
  • 🔔 إعلانات من الإدارة

📋 المتطلبات الأساسية

  • حساب Google (مجاني)
  • الوصول لإدارة تطبيق نواقص على HF Spaces
  • صلاحية لتعديل ملفات المشروع

1 إنشاء مشروع Firebase

1.1 اذهب إلى Firebase Console
1.2 اضغط "Add Project" (إضافة مشروع)
1.3 اكتب اسم المشروع: nawaqes-app
1.4 اضغط Continue → عطّل Google Analytics → اضغط Create Project
1.5 انتظر 30 ثانية حتى يُنشأ المشروع، ثم اضغط Continue

2 إضافة تطبيق Web (للإشعارات في المتصفح)

2.1 من الصفحة الرئيسية للمشروع، اضغط أيقونة </> (Web)
2.2 App nickname: Nawaqes Web → اضغط Register app
2.3 ستظهر لك firebaseConfig — انسخها (ستحتاجها لاحقاً):
const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXX", authDomain: "nawaqes-app.firebaseapp.com", projectId: "nawaqes-app", storageBucket: "nawaqes-app.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };

3 تفعيل Cloud Messaging وإنشاء VAPID Key

3.1 من القائمة الجانبية: Engage → Cloud Messaging
3.2 تأكد أن Firebase Cloud Messaging API (V1) مفعّل (افتراضياً مفعّل للمشاريع الجديدة)
3.3 اذهب إلى تبويب Web Configuration
3.4 تحت Web Push certificates، اضغط Generate key pair
3.5 انسخ الـ VAPID key (يبدأ بـ BJ...)

4 إضافة تطبيق Android (للإشعارات في APK)

4.1 من إعدادات المشروع: Project Settings ⚙️ → Add app → Android
4.2 املأ البيانات:
  • Android package name: com.nawaqes.app
  • App nickname: Nawaqes
4.3 اضغط Register app
4.4 حمّل ملف google-services.json

5 إعداد Service Account (لإرسال الإشعارات من السيرفر)

5.1 اذهب إلى Project Settings → Service Accounts
5.2 اضغط Generate new private key → Generate key
5.3 سيُحمّل ملف JSON — احفظه بأمان (يحتوي صلاحيات إدارة المشروع)

6 إضافة المتغيرات على Hugging Face Spaces

6.1 اذهب إلى: إعدادات الـ Space
6.2 ابحث عن قسم "Variables and secrets"
6.3 أضف المتغيرات التالية (استبدل القيم بالتي حصلت عليها):
FIREBASE_PROJECT_ID = nawaqes-app FIREBASE_API_KEY = AIzaSyXXXXXXXXXXXX FIREBASE_AUTH_DOMAIN = nawaqes-app.firebaseapp.com FIREBASE_STORAGE_BUCKET = nawaqes-app.appspot.com FIREBASE_MESSAGING_SENDER_ID = 123456789012 FIREBASE_APP_ID = 1:123456789012:web:abcdef123456 FIREBASE_VAPID_KEY = BJXXXXXXXXXXXX
6.4 اضغط Save — سيُعاد بناء السيرفر تلقائياً

7 تحديث APK بإضافة Firebase

7.1 انسخ ملف google-services.json إلى مجلد المشروع:
/home/z/my-project/nawaqes-apk/webview-apk/app/google-services.json
7.2 أعد بناء APK:
cd /home/z/my-project/nawaqes-apk/webview-apk ./gradlew assembleRelease
7.3 ارفع APK الجديد على السيرفر:
cp app/build/outputs/apk/release/app-release.apk \ /home/z/my-project/nawaqes/downloads/nawaqes-v3.1.0-fcm.apk git add downloads/nawaqes-v3.1.0-fcm.apk git commit -m "Update APK with Firebase" git push

8 اختبار الإشعارات

8.1 افتح التطبيق: https://safwatkhokha-nawaqes.hf.space
8.2 سجّل دخولك → اذهب للإعدادات → فعّل الإشعارات
8.3 اسمح بالإشعارات من المتصفح/النظام
8.4 افتح Console المتصفح (F12) وستجد: [FCM] Web device token: xxxx
8.5 اختبر الإرسال عبر سكريبت الاختبار:
node /home/z/my-project/nawaqes-apk/firebase/test-push.js YOUR_DEVICE_TOKEN
⚠️ تنبيه أمني:
  • لا تشارك ملف google-services.json أو Service Account JSON علناً
  • لا ترفع Service Account JSON على Git — استخدم HF Secrets فقط
  • الـ VAPID key آمن للنشر في الكود (وليس سرّاً)
